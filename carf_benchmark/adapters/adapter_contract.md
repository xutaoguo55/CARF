# CARF-Benchmark Adapter Contract

Adapters convert native model outputs into the CARF-Benchmark v1
`method_scores.csv` schema. They should not compute CARF metrics themselves;
the benchmark runner owns metric computation.

## Active Adapter CLI

The shared adapter entrypoint is:

```bash
python3 carf_benchmark/adapters/run_adapter.py <command> [options]
```

Active model IDs:

- `scgpt`
- `scfoundation`
- `uce`
- `scbert`
- `gears`
- `cpa`
- `mean_baseline`

The adapter refuses to fabricate output. It requires a native model artifact,
then writes CARF-compliant rows with ranks recomputed within that model's
actual gene scope.

## Native Output Modes

Gene-level native outputs:

```bash
python3 carf_benchmark/adapters/run_adapter.py gene-score-csv \
  --model-id scgpt \
  --dataset-id adamson_2016_perturbseq \
  --perturbation-id DDIT3 \
  --native-output scgpt_DDIT3_gene_scores.csv \
  --gene-column gene_symbol \
  --score-column predicted_delta \
  --output carf_benchmark/runs/adamson_2016_perturbseq/adapters/scgpt_DDIT3.csv
```

Predicted-expression native outputs, typical for GEARS/CPA style runs:

```bash
python3 carf_benchmark/adapters/run_adapter.py expression-delta \
  --model-id gears \
  --dataset-id norman_2019_combo \
  --perturbation-id AHR_FEV \
  --predicted-expression gears_AHR_FEV_predicted_expression.csv \
  --baseline-expression control_expression.csv \
  --output carf_benchmark/runs/norman_2019_combo/adapters/gears_AHR_FEV.csv
```

Append adapter output to a run:

```bash
python3 carf_benchmark/adapters/run_adapter.py append \
  --run-method-scores carf_benchmark/runs/norman_2019_combo/inputs/method_scores.csv \
  --adapter-output carf_benchmark/runs/norman_2019_combo/adapters/gears_AHR_FEV.csv \
  --output carf_benchmark/runs/norman_2019_combo/inputs/method_scores.csv
```

## Required Output

Each adapter writes one row per scored gene:

| column | meaning |
| --- | --- |
| `dataset_id` | benchmark dataset identifier |
| `perturbation_id` | perturbation identifier |
| `model_id` | stable model id from `registry/models.csv` |
| `model_name` | display name |
| `gene_symbol` | HGNC gene symbol when available |
| `score` | signed native score when meaningful |
| `score_abs` | non-negative magnitude used for ranking |
| `rank` | 1 means strongest predicted perturbation |
| `scope_included` | whether this gene is in the model's evaluable scope |
| `score_type` | score class, e.g. `cosine_shift` or `predicted_delta` |
| `source_file` | native file used to produce the row |

## Active v1 Adapters

- `scgpt`: convert scGPT gene-score or predicted-expression outputs
- `scfoundation`: convert scFoundation gene-score or predicted-expression outputs
- `uce`: convert UCE gene-score or decoder outputs
- `scbert`: convert scBERT gene-score or decoder outputs
- `gears`: convert GEARS predicted-expression outputs
- `cpa`: convert CPA predicted-expression outputs
- `mean_baseline`: generated by public Perturb-seq input preparation

All adapters should keep the original native prediction files alongside the
standardized CSV and record their path in `source_file`.
